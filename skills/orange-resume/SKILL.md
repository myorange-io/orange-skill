# Orange Build — Resume

세션이 끊겼거나 시간이 지난 뒤 프로젝트를 이어갈 때, **지금 어디까지 왔는지** 빠르게 파악한다.
토큰을 거의 쓰지 않는 가벼운 현황 점검이다.

## 1. 현재 상태 읽기

현재 폴더에서 다음을 확인한다:

- `PLAN.md` — 있으면 전체를 읽는다. 특히 `## 진행 상황` 체크리스트.
- `MEMORY.md` — 있으면 **마지막 항목 한두 개만** 읽는다. git log보다 풍부한 "여태 무엇을·왜
  했나" 맥락이 담겨 있다 (전체를 다 읽지 않는다 — 토큰 절약).
- `git log --oneline -12` — 최근 작업 기록.
- `package.json` 존재 여부 — 앱이 scaffold 됐는지.
- `design/` 폴더 — Stitch 디자인이 있는지.

`PLAN.md`가 **없으면**: 아직 시작 전이다. "아직 기획 전이에요 — `/orange-start`로
시작하세요." 라고 안내하고 끝낸다.

## 2. 현황 알려주기

`PLAN.md`가 있으면 짧게 정리해 보여준다 (길게 늘어놓지 않는다):

```
📋 [앱 이름]
   완료: [끝난 항목들]
   다음: [체크리스트의 다음 미완료 항목]
   라이브: [배포됐으면 URL, 아니면 "아직 배포 전"]
```

다음 단계를 한 줄로 짚어준다. 예: "다음은 '신청 목록' 화면 구현이에요."

`MEMORY.md`가 있으면 마지막 기록을 한 줄로 곁들인다. 예: "지난번엔 로그인 없이 가기로 정했어요."

## 2-1. 이어가기 기록 (선택)

`MEMORY.md`가 있으면, 끊겼다 다시 잇는 흐름도 기록으로 남길 수 있다 — 파일 끝에 항목을
**덧붙인다**(append, 덮어쓰지 않는다):

```markdown
### [YYYY-MM-DD] 이어가기
- **멈췄던 곳**: [마지막으로 끝낸 단계/화면]
- **왜 멈췄나**: [사용량 소진·시간·막힘 등. 막혀서 멈췄다면 무엇 때문이었는지]
- **다음에 할 것**: [체크리스트의 다음 항목]
- **돌아와 한 것**: [다시 이어 무엇을 어떻게 진행했는지 — 이어간 뒤 채운다]
```

날짜는 `date +%Y-%m-%d`로 얻는다. 자세한 규칙은
`../orange-start/references/memory-log.md` 참고. (기록을 원치 않으면 건너뛴다.)

## 3. 이어가기

사용자에게 `/orange-start`를 실행하면 멈춘 지점부터 자동으로 이어진다고 안내한다.
(`orange-start`가 폴더 상태를 보고 알맞은 단계로 진입한다.)
